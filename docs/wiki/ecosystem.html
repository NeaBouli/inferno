The Self-Reinforcing Loop

1. The Loop

Every action in the IFR protocol reinforces the loop. Contributors lock, Builders borrow, Users buy — and every step creates buy pressure on Uniswap.

Contributor 100M IFR locked
→
CommitmentVault Tranche-Lock
+

↓

→

→

↓

→

→

→

→

↻ LOOP

2. The 3 Principles

Principle 1: Tokens Prove Commitment

  • Locked tokens = on-chain truth, no trust required
  • Smart contract enforces it, community can verify

Principle 2: Locked Tokens Work

  • Tokens that are locked and forgotten = wasted
  • CommitmentVault + LendingVault make them productive

Principle 3: Every Action Creates Buy Pressure

  • Builder borrows → must repay → buys on Uniswap
  • Builder cannot repay → liquidation → buys on Uniswap
  • User wants discount → buys IFR to lock
  • Interest → deeper LP → stable price

3. All Stakeholders

4. Pricing Model

Phase 1–2 (now): Lock Model

User buys IFR, locks permanently, lifetime access.

At P0 = $0.0000002: 5,000 IFR cost $0.001

Phase 3+ (when price is right): Payment Model

Threshold: P0 × 2,500 ($0.0005)

User pays X IFR per month directly to Builder.

X = Price_in_USD / IFR_TWAP_Price

USD-Equivalent Lock

Builder says: “Lock IFR worth $10.”

Smart Contract: current price × locked amount ≥ $10 → access granted.
