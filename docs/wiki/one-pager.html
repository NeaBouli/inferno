Inferno $IFR — One Page Overview

Everything you need to know about Inferno in one page.

What is Inferno?

Inferno ($IFR) is a deflationary ERC-20 utility token on Ethereum. Every transfer permanently burns a portion of the tokens, continuously reducing the total supply. Users lock IFR on-chain to unlock lifetime premium access across a growing network of partner products and services. There is no presale, no VC funding, and no mint function — launched via a Community Fair Launch Model (CFLM).

Tokenomics

  • Burn-on-transfer: 2.5% of every transfer is permanently burned (2% sender + 0.5% recipient), making IFR continuously deflationary.
  • Buyback mechanism: BuybackVault collects ETH and buys IFR on Uniswap V2, splitting 50/50 between burn and treasury.
  • Lock = Tier: Lock IFR in the IFRLock contract to unlock Bronze / Silver / Gold / Platinum access across all partner products.
  • Builder rewards: PartnerVault distributes IFR to builders when users lock through their integrations — 4% of supply reserved.

Products

  1. Benefits Network — merchants verify IFR lock status to offer real-world discounts. Next.js 14 frontend + Express backend with PWA support.
  2. Creator Gateway — content creators integrate IFR locks to gate premium content. Lock-triggered rewards flow to creators from the PartnerVault.
  3. AI Copilot — an AI assistant trained on Inferno protocol documentation. Answers questions about tokenomics, contracts, and integration.
  4. Points System — gamified engagement layer with SIWE authentication. Users earn points for on-chain activity and protocol participation.
  5. Governance Dashboard — visualize and interact with governance proposals, timelock status, and parameter changes in real time.

Technology

  • Ethereum Mainnet — deployed and verified on March 5, 2026.
  • Solidity 0.8.20 + OpenZeppelin v5 — battle-tested base contracts with modern security patterns.
  • Slither audit + 556 tests — zero High/Critical findings, 99% statement coverage.
  • 48h Timelock + Guardian — all parameter changes are delayed and publicly visible before execution.
