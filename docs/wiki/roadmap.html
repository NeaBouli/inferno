Roadmap

Five-phase development plan from foundation to full DAO governance.

Current status: Phase 4 (Mainnet Launch) — 14 on-chain components deployed and verified on Ethereum Mainnet (March 2026). Token distribution complete (1B IFR, CFLM). Ownership transferred to Governance. Next milestones: LP pairing, LP lock, Governance owner → Multisig.

Phase 1: Foundation Done

Q4 2025 — Core protocol contracts and testing infrastructure.

  • InfernoToken (ERC-20, 9 decimals, fee-on-transfer)
  • LiquidityReserve (6-month lock, 50M/quarter)
  • Vesting (12-month cliff, 36-month linear)
  • BuybackVault (50/50 split, 60-day activation)
  • BurnReserve (permanent burn + tracking)
  • Hardhat test suite (core contracts)
  • Slither security audit (0 high/critical)

Phase 2: Governance & Lock Done

Q1 2026 — On-chain governance, utility lock, Sepolia deployment.

  • Governance contract (48h timelock, propose/execute/cancel)
  • IFRLock contract (lock/unlock, isLocked(), multi-app lockType)
  • Sepolia deployment (14 contracts, all verified)
  • Uniswap V2 LP pairing (IFR/WETH)
  • Ownership transfer to Governance
  • Governance Proposals #0 executed, #1–#3 cancelled, #4+#5 executed, #6 executed 13.03.2026
  • Token Dashboard (React 18 + Vite)
  • Governance Dashboard (React 18 + TypeScript + Tailwind)
  • 556 total tests (521 protocol + 35 ecosystem)

Phase 3: Ecosystem In Progress

Q1–Q2 2026 — Builder ecosystem, applications, audit preparation.

  • PartnerVault v2 (authorizedCaller, anti-double-count, algo throttle)
  • FeeRouterV1 (EIP-712 vouchers, protocol fee routing)
  • Benefits Network (QR-based lock verification)
  • Creator Gateway (YouTube × IFR Lock Bridge)
  • Points Backend (SIWE auth, EIP-712 voucher issuance)
  • AI Copilot (RAG knowledge base, 3 modes)
  • Builder Directory (filter by type/status)
  • BuilderRegistry.sol (on-chain builder registry, 27 tests, deploy script ready)
  • 3-Tier Wallet Verification (Signer / Voter / Builder)
  • On-Chain Reader (IFRLock + Gnosis Safe getOwners() auto-sync)
  • verify.html + Bot /verify /mystatus commands
  • Topic Access Guards (Core Dev, Council, Vote, Dev&Builder)
  • 25 wiki pages, transparency report, SEO canonical tags
  • Lock UI in Token Dashboard
  • Ownership Transfer (v2 redeploy)
  • BuilderRegistry Mainnet deploy (after audit)
  • LP Token Lock/Burn
  • Professional security audit

WalletConnect Integration

The IFR WalletConnect integration is being rolled out in phases, progressively unlocking features as the ecosystem matures.

✅ Phase 1 — Core Infrastructure (March 2026)

  • ✅ wallet-core.js — Central session manager (connect/disconnect/auto-reconnect)
  • ✅ ifr-state.js — On-chain state reader (IFR balance, lock status, bootstrap contribution)
  • ✅ Bootstrap Widget — contribute() / claim() / refund() fully functional
  • ✅ Landing Page — Connect button in header with wallet dropdown
  • ✅ Agent Page — Free/Premium tier display based on IFRLock on-chain check

⏳ Phase 2 — Personalization (after Bootstrap Start, April 2026)

  • ⏳ Copilot receives wallet address + tier via URL params → personalized responses
  • ⏳ “Buy IFR” button → Uniswap (only after LP live, July 2026+)
  • ⏳ Live IFR token price display (only after LP live)
  • ⏳ Bootstrap countdown widget fully active

🔒 Phase 3 — Full Ecosystem (after LP live, Q3 2026+)

  • 🔒 AI Copilot Gate — ≥1,000 IFR locked → gated content platform
  • ✅ Builder Tools — BuilderRegistry.sol deployed, on-chain isBuilder() check integrated in Telegram Bot
  • 🔒 Telegram Wallet Whitelist — WalletConnect → IFRLock check → Topic access
  • 🔒 Governance Tools — Core Team whitelist → Proposal creation/voting UI
  • 🔒 Creator Gateway — OAuth + WalletConnect + IFRLock → content monetization

Stack: ethers.js v5 + MetaMask + IFRLock ABI. All wallet interactions are non-custodial — IFR remains on-chain at all times.
